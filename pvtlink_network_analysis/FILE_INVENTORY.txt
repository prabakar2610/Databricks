PROBLEM:
  Customer setup Azure Private Link to connect Databricks to internal
  services behind a load balancer, but DNS resolution returns PUBLIC IP
  instead of PRIVATE IP, preventing connectivity.

SOLUTION:
  This suite provides:
  ✅ Diagnostic scripts to identify root cause
  ✅ Step-by-step troubleshooting guides
  ✅ Specific fixes for each issue type
  ✅ Comparison testing (Databricks vs Azure VM)
  ✅ Configuration validation
  ✅ Result interpretation tools

OUTCOME:
  Quickly identify and fix:
  • NCC misconfiguration (most common)
  • Private DNS Zone issues
  • Load Balancer problems
  • NSG rule issues
  • Backend service problems
